Splay

Splay是一种基于均摊时间复杂度的平衡树

  • 不能保证任何时刻这棵二叉查找树都长得非常均衡
  • 能保证如果进行 $m(m\rightarrow +\infty)$ 次操作总时间复杂度趋近于 $O(mlogn)$，均摊到单次操作的时间复杂度为 $O(logn)$

Splay的核心思想是通过旋转操作进行自我调整，从而获得均摊较低的时间复杂度

Splay的旋转

一次旋转操作，改变Splay的一个局部的结构，但不影响Splay作为二叉查找树的性质

我们可以把一次旋转视作一个函数 void rotate(int x)，该函数尝试将 $x$ 和其父节点 $fa(x)$ 进行倒反天罡，即让 $x$ 变成 $fa(x)$ 的父节点

Splay

设 $x$ 是当前待旋转的节点，$y$ 是 $x$ 的父亲，$z$ 是 $y$ 的父亲

如果 $y$ 比 $x$ 小，那么 $y$ 应该顶替到 $x$ 的左子树上，如果 $y$ 比 $x$ 大，那么 $y$ 应该被顶替到 $x$ 的右子树上，为了方便叙述，我们不妨说 $y$ 将要顶替 $x$ 的 $T$ 子树，$T\in \{左，右\}$

  1. 第一步，通知 $z$ 它的儿子变了
    • 如果 $z$ 的左儿子是 $y$，那么 $z$ 的左儿子应该改为 $x$
    • 如果 $z$ 的右儿子是 $y$，那么 $z$ 的右儿子应该改为 $x$
  2. 第二步，通知 $x$ 的父节点变了，让 $x$ 的父节点是 $z$
  3. 第三步，把 $x$ 的 $T$ 子树打包丢给 $y$，现在 $x$ 的 $T$ 子树就空出来了，并且 $y$ 到 $x$ 的联系也被彻底切断了，$x$ 的 $T$ 子树占据了原本 $x$ 在 $y$ 眼中的地位
  4. 第四步，把 $y$ 挪到 $x$ 的 $T$ 子树上
  5. 最后可能会需要用 void pushUp(int x) 来做一些区间信息的维护统计

最终，我们得到了 $rotate$ 这么一个非常有用的小工具，它可以帮我们改变二叉查找树的形态，但不改变二叉查找树的性质

不过像上述代码这么写时间复杂度是没有保障的 （但只要题目没有用心出数据似乎都能过）

Splay

研究发现，对于一棵大小为 $m$ 的二叉查找树，使用 $m$ 次 $Splay$ 函数，无论输入的数据 $x$ 是什么，以什么样的顺序出现，它的均摊复杂度都是 $O(mlogn)$

Splay

现在我们拥有了

  • void rotate(int x, int & f)：用于将 $x$ 和自己的父节点 $fa(x)$ 倒反天罡
  • void Splay(int x, int & f)：用于将 $x$ 旋转到 $f$ 上去，$f$ 可能是树根，可能是某元素的子节点

乍看之下这两个函数平平无奇，但实际上我们已经能用它们做一切区间操作了，包括区间插入，区间查询，区间删除，区间提取

同时一切在线段树中接触过的懒标记思想，可以无痛迁移到Splay上来

同时Splay还比线段树更灵活，更动态

唯一缺点就是常数大

单点插入 void insert(int k, int v)

首先定位到要插入的位置 $x$，假设是要插入在 $x$ 的 $T$ 子树上（$T\in \{左，右\}$），然后新建一个节点 $y$，$y$ 的 $key$ 值为 $k$，$value$ 值为 $v$

然后将 $y$ 旋转到根上来，这一步一定要做，这是保证Splay时间复杂度正确的关键

查询前驱/后继 int findPrev/findNext(int x)

首先把 $x$ 旋转到树根，然后

  • 如果是查 $x$ 的前驱，那么从根出发，先往左边走一步，再一直往右走，直到走不了为止
  • 如果是查 $x$ 的后继，那么从根出发，先往右边走一步，再一直往左走，直到走不了为止

为了避免特判很多特殊情况，我们通常会添加两个哨兵，一个哨兵取值 $-\infty$，一个哨兵取值 $+\infty$，从而可以保证除哨兵外每个节点一定是有前驱和后继的

查询前驱 int findPrev(int x)

例如我们要定位 $6$ 号节点的前驱节点编号

插入到某节点之后 void insertAfter(int x, int v)

首先把 $x$ 旋转到根，再把 $x$ 的后继 $y$ 旋转到根的右儿子上，此时 $y$ 的左儿子一定为空

然后新建一个节点 $z$，把 $z$ 挂在 $y$ 的左儿子上，并进行 pushUp 就行了

插入到某节点之后 void insertAfter(int x, int v)

例如我们要插入一个新节点到 $5$ 号节点之后

找到第 $k$ 个元素 int findKth(int k)

我们先假设Splay上有哨兵，并且值为 $-\infty$ 的那个哨兵为第 $1$ 个元素（之后的讨论都是基于此假设）

为了支持这个功能，Splay上每个节点 $u$ 需要维护自己子树的大小 $siz(u)$

提取一段区间 int extract(int l, int r)

我们定位到第 $l-1$ 个元素 $a$ 和第 $r+1$ 个元素 $b$

先将 $a$ 旋转到根的位置，再让 $b$ 旋转到根的右儿子上

提取一段区间 int extract(int l, int r)

例如我们要提取第 $3$ 个节点到第 $6$ 个节点这段区间

区间求和sum(l, r)

我们把 $[l,r]$ 这段区间提取出来，然后直接调用在 pushUp 中维护好的区间和就行了

区间删除erase(l, r)

我们把 $[l,r]$ 这段区间提取出来，然后把它和它的父节点断开就行了

还有很多操作

大家其实应该能看出这些操作的套路了

我们就是用 void Splay(int x, int & f) 这个函数，通过调整点的位置，把我们想要的单点或区间给提取出来，一般来说我们会把目标单点放在根上去，把目标区间放在靠近根的位置上去

Splay上的每一个节点代表的都是一个区间，区间包含的元素就是其子树内的所有元素
