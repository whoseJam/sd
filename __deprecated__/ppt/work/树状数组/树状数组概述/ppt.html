树状数组

动态连续和查询问题

给定一个 $n$ 个元素的数组 $\{A_i\}$，你的任务是设计一种数据结构，支持以下两种操作：

  1. Add(x, d)：让 $A_x$ 的数值增加 $d$
  2. Query(l, r)：求 $A_l+A_{l+1}+...+A_r$ 的值

假设总共有 $m$ 次操作，如何才能让 Add 和 Query 都快速地完成呢？

分析

如果直接用数组模拟，那么 Add 可以 $O(1)$ 完成，Query 是 $O(n)$ 的，总复杂度为 $O(nm)$

分析

如果另外维护一个前缀和数组 $\{S\}$ 来存储 $A_1$ 到 $A_n$ 的和，那么 Query 可以 $O(1)$ 完成，Add 是 $O(n)$ 的，总时间复杂度还是 $O(nm)$

分析

时间复杂度 直接数组模拟 维护前缀和
Query 时间复杂度 $O(n)$ $O(1)$
Add 时间复杂度 $O(1)$ $O(n)$

一个自然的思路就是，我们能否结合上面两者的优点，维护部分前缀和，在修改和查询之间保持某种微妙的平衡

二进制拆分

我们尝试对每一种长度的前缀和做二进制拆分......

  • 对于 Query 类操作来说，任意一个前缀和一定能被 $O(logn)$ 算出
  • 对于 Add 类操作来说，任意一个位置最多被 $O(logn)$ 个区间覆盖

可以发现我有意把有效区间排列为树形结构，而有效区间又恰好有 $n$ 个，是一个数组能放下的东西，这玩意就是树状数组

这是一份长度为 $12$ 的序列的树状数组的情况

lowbit

对于正整数 $x$，我们定义 $lowbit(x)$ 为 $x$ 的二进制表达式中最右边的 $1$对应的值

  • $24=(11000)_2$，则 $lowbit(24)=(1000)_2=8$
  • $18=(10010)_2$，则 $lowbit(18)=(10)_2=2$

在程序中，我们可以这么来计算 $lowbit(x)$
