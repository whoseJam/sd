Math Gallery

Visual walkthrough of sd.Math contracts.

→ / space to advance. Animated slides also use space to trigger. LaTeX is rendered through MathJax3.

defaults

Black x^2 + y^2 = z^2 at center, fontSize 60. MathJax produces the glyphs; sd.Math wraps them.

font-size

Same equation x^2 at 30 / 60 / 120 px stacked top→bottom — each ~2× taller than the previous.

anchor

Three red dots, three labels pinned by different anchor args. Same contract as sd.Text: (x,y) = bottom-left, (cx,cy) = middle-left, (centerX,centerY) = box center.

subtext-fill

In x^2 + 2x + 1, color the first x red (i=0) and the second blue (i=1). Verifies that setSubtextFill matches MathJax-rendered glyphs, not the LaTeX source.

set-text-default

Press space — x → x^2 with default mapping. The single source x is duplicated (buildMapping(1,2)) — one copy morphs to target x, another morphs to ².

set-text-equal-counts

Press space — a+b → a-b, default mapping. Source/target have the same glyph count (3), so buildMapping pairs them 1-to-1: a→a, +→-, b→b. Cleanest default behavior. (Explicit string mappings on Math need an engine fix — patterns must match MathJax-rendered glyphs, not LaTeX source.)

set-font-size-animated

Press space — fontSize 24 → 80 on ∫₀^∞ e^(-x²) dx. Bottom-left corner stays glued to the red dot through the tween.

set-center-and-font-size

Press space — fontSize 30 → 100, chained with .setCenterX(0).setCenterY(0). The box center should stay pinned to the red dot. (Suspected bug: Math may drift because setX/setY trigger on the inner html, decoupled from this.attributes.)

set-text-pythagoras

Press space — a² + b² → c², default mapping. buildMapping(5, 2) distributes the 5 source glyphs across 2 target glyphs.
